Recomendaciones generales de seguridad

  • Control de acceso: Verificar que toda función privilegiada (mint, pausas, upgrades, cambio de parámetros) está protegida y, idealmente, tras un multisig/timelock.
  • Testing de invariantes: Añadir tests de propiedades (fuzzing/invariant testing con Foundry) sobre las invariantes económicas clave (conservación de fondos, solvencia, exchange rate).
  • Oráculos: Si se consumen precios externos, validar frescura (staleness) y cotas de desviación; considerar fuentes redundantes.
  • Reentrancy y patrones checks-effects-interactions: Confirmar guardas de reentrada en toda función que mueva fondos o llame a contratos externos.
  • Seguridad de upgrades: Si el sistema es actualizable, revisar colisiones de storage y proteger el proceso de upgrade tras gobernanza/timelock.
  • Monitoreo continuo: El código cambia: recomendamos revisión automática ante cada deploy/upgrade (nuestro servicio de monitoreo continuo cubre exactamente esto).
